# AIDA 用户手册

本文档介绍如何在 AIDA 平台开展安全评估工作。其核心工作流如下：

**创建 Assessment项目 → 自动配置与了解 Workspace → 连接 AI → 驱动命令执行与审批 → 查看 Findings/Recon 等沉淀结果。**

---

## 1. 启动评估任务：创建 Assessment

进入本地前端界面的 Dashboard 面板，并点击 **"New Assessment"** 唤醒表单，相关的定义元数据将在随后指引 AI 的行为范式。

关键输入字段说明：
- **Assessment Name**: 此测试项目的独立标题名称（例如：`Internal_Network_Test`）。
- **Target Domains / IP Scopes**: 限定且被许可进行主动探测的特定 URL 域、网段地址等范围界限。
- **Scope & Objectives**: 给测试代理的评估范围阐述，包括不能发包等外部环境制约准则等。

完成基本情况录入提交后，平台将生成独立的项目空间进行跟踪管理。

---

## 2. 理解 Workspace (工作区隔离与结构)

生成 Assessment 项目后系统后台随之为该用例划分出完全隔离的底层挂载文件目录实体。按默认配给形式位于系统宿主机 `~/.aida/workspaces/<项目名>/` 下。

其典型基础层级树包含：
```text
/workspace/<assessment-name>/
├── recon/      # 探测环节的数据输出流向
├── exploits/   # 用于尝试性突破复试的探针或脚本
├── loot/       # 如数据抽取结果、核心转储之类敏感证明产物存放区
├── notes/      # 其他不归类手签札记存放点
├── scripts/    # 用户自研外入辅助脚本载体
└── context/    # 前端附件模块所转存的用户补充源材料
```
此结构支撑起了上下文连贯能力。不论项目经历过多少次休眠复开，执行命令依旧基于默认项目基准路径展开，并自动进行存单。

---

## 3. 信息补全与上下文引入

### Context 文档上传
若针对当前所评估结构持有内部协议释意、API 预定义图纸（如 OpenAPI 格式）、架构系统图例或是往期诊断文献此类白盒级别指导性明细。建议于对应 Assessment 的 **Context** 模块提前上传，实体文件会推送至 `context/` 存储块，AI 则能够借此获得更高维度的全景认知。

### Recon 数据与预扫描导入
在大体量及复杂架构面前通过手动或者现成的大型组件预制收集好如：基于离线的 Nmap 解析成果、基于 Nuclei 编排出的初期缺陷清单和子系统枚举资产。可直接将 JSON 与 XML 数据集由 Web 前端向 `Recon` 面板进行输入集成。AI 被唤醒连通那一刻即加载上述资料构建策略起步。

### 凭据系统与占位符应用 (Credentials Placeholders)
通常带有一定侵入授权的应用场景均需要诸多合法前置要素或强 Token 以构建回程凭证，但避免原生敏感信息直抛于公开命令集是核心安全需要。通过前端 Credentials 服务功能生成相应的凭签实体可以获得一组转录代名。
- 例设定表参变量：`USER_API_SECRET`
- 体系提供映射位：`{{USER_API_SECRET}}`

如此一来 AI 决定以 API 测试模式驱动工具（如：`curl -H "Auth: Bearer {{USER_API_SECRET}}" ...`）时系统将自动在下沉至容器真正动作前替换完实体密码，截绝泄露可能。

---

## 4. 连接 AI 并执行控制

一切铺垫妥当之后即可启动项目关联流程，强烈推荐使用项目直属配套： `aida.py`

```bash
python3 aida.py --assessment "Internal_Network_Test"
```

该工具会自动寻址出相匹配的如 Claude Code 或者 Kimi 后将其起吊。顺势进行评估级 Prompt 同步装订。建立完毕开始使用平台供给的能力接口。AI 可自由选用工具对所指靶群进行直接测试交火。所有工具行为均在预设隔壁容器发起。

---

## 5. Command Approval (命令审批系统模式)

出于风险规避需求，AI 发往系统执行的所有终端类操作均接受平台拦截鉴定。相关调度管控位于界面菜单的 **Settings -> Command Execution**：

1. **Closed (全阻断模式)**：强风控场景。一切不分种类来源的执行动作一经拦截立马上报前端看板弹窗以待人员主观核实与授权点击 (Approve)，如果在无操作状态下等候超时指令默认丢弃报错。
2. **Filtered (关键词过滤模式)**：自动判定且兼具防幻觉功能的折中选项。配合在后台罗列禁区词库设置（诸如强制降权 `drop` 数据库动作或清理 `rm -rf` 指令群）。平时命令放量快速推进，一旦越过设防标尺即转为人工悬挂待受。
3. **Open (全开放模式)**：针对靶机系统纯演习类的最佳通配，毫无阻断流控制操作。下即生效。

审批过的命令行流及其后续吐露的控制台回应 `stdout/stderr`均于历史回溯处详细存留可复查比对。

---

## 6. 结果与发现：Findings 与 Cards 系统

当安全事件触达实质性突破边界或是值得关切的数据被发掘，AI 会调用相应模组制作结构化证明输出为各体系等级评点的 Cards 数据条落座于平台前端显示列。

- **Findings (漏洞通报记录)**：具备明显损害特性判定。将标记：致因来源、危害划分度 (Severity)、核实进展状态以及证明该项成立的关联指令流或是复演动作过程证据链。
- **Observations / Info (通用记录标签)**：未见明显越权的资产版本流出暴露或配置不规范的记录备注，提供架构情报整理素材。

在此基础架构下，您也可以直接手工调阅 Cards 修改其标注或是为测试后处理环节查漏补缺。
